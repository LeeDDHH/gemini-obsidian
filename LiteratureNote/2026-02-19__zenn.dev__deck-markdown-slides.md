## TL;DR

- `deck`：Markdownで書いたコンテンツをGoogle Slidesに変換するOSSツール
- 「コンテンツ（Markdown）」と「デザイン（Google Slides）」を分離するアプローチ
- Gitでバージョン管理しながらスライド作成できる
- Homebrew でインストール可能、Google API認証設定が必要

## 要約

著者 katayan（daisuke23）が、Markdownを使ってGoogle Slidesを作成できるツール `deck` の導入方法を解説した記事（2025年8月27日）。

### deckの特徴

- **コンテンツとデザインの分離**: Markdownでコンテンツを書き、Google Slidesでデザインを適用
- **Gitバージョン管理**: テキストベースなのでdiffが取れる、チームでの共同編集も可能
- **継続的なスライド修正**: `deck apply` で随時反映、Watch モードでリアルタイム同期

### 導入手順

1. **インストール**: `brew install deck`
2. **Google Cloud Console**: プロジェクト作成 → Google Slides API + Google Drive API を有効化
3. **OAuth認証**: デスクトップアプリとしてOAuthクライアントIDを設定
4. **認証ファイル配置**: ダウンロードした JSON ファイルを所定のディレクトリに配置

### 基本的な使い方

```markdown
---
スライド1の内容
---
スライド2の内容
```

- `---` でスライドを区切る
- `deck apply` で Google Slides に反映
- Watch モードで編集がリアルタイム同期

## 核心ポイント

- **Marpとの比較**: Marpはローカルでの静的HTML/PDF出力。deck はGoogle Slidesとの連携が強み
- **チーム共有**: Google Slidesなので作成後の共有・コメントが容易
- **環境設定コスト**: Google API設定が最初だけ必要だが、一度設定すれば快適

## 内 적용 아이디어

- テックブログや社内LTのスライドをMarkdownで書くようにすると管理しやすくなる
- PR説明や設計資料のスライド版を素早く作るのに使えそう
- Obsidian で書いたメモ → deck でスライド化というフローが作れる可能性
