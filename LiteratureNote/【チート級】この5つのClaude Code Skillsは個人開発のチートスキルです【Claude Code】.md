# 【チート級】この5つのClaude Code Skillsは個人開発のチートスキルです【Claude Code】
https://www.youtube.com/watch?v=gkcUAtAfw1I

了解です。貼ってくれた内容は、**「Claude Code Skills（skills.sh）で“プロの知見”をプロジェクトに持ち込む方法」**を、実演ベースで紹介する動画の台本（/文字起こし）ですね。要点をギュッと整理します。

  

## **この動画の主旨（何を伝えたいか）**

- Claude Code の **Skills** を使うと、UI/UX改善・ベストプラクティス点検・ファネル分析などを“専門家の型”として導入できる
    
- skills.sh（ランキング）からインストールして、**ローカル/グローバルで使い回せる**
    
- 個人開発の売上（例：月5万円）にも実際に効いている、というストーリー
    

  

## **動画内で紹介しているスキル（流れ順）**

1. **Find Skills（検索用スキル）**
    
    - 「こういう用途のスキルない？」を投げると、該当スキルを探して提案してくれる
        
    - スキルが自動起動しないこともあるので、明示的に呼ぶ運用も紹介
        
    
2. **UIUXProMax（UI/UXの参照→反映）**
    
    - 参考にしたいページURLを渡すと、タイポグラフィ/スタイルを抽出して
        
        それっぽいUIを生成（HTML/CSS/Tailwind等）
        
    - 著作権配慮として「そのままコピペではなくアレンジ推奨」も言及
        
    
3. **Vercel React/Next.js Best Practices（ベストプラクティス監査）**
    
    - プロジェクト全体を見て、Next/Reactのアンチパターンや最適化漏れを指摘
        
    - 例：jszip をトップレベルimportしていて **バンドル肥大 → dynamic import推奨**
        
    - Promise.allでウォーターフォール回避、Suspense境界などの観点も
        
    
4. **Remotion Best Practices（動画生成の型）**
    
    - Remotionで「アプリ紹介動画」をコード生成
        
    - npxで起動→プレビュー→レンダー出力、までを一気通貫で実演
        
    
5. **ファネル分析スキル（自作）＋CRO系スキル（例：Kuro Methodology）**
    
    - GA4 / Supabase 等と組み合わせ、LP→登録→初回実行→課金の離脱率を可視化
        
    - SNS流入を除外するなど、ノイズ除去の考え方も
        
    - 「売上2倍」を、単一レバーではなく **複数レバーの掛け算**で作る、という方針
        
    

  

## **動画が暗に言っている“使い方のコツ”**

- Skillsは「便利ツール」じゃなくて **専門知識のパッケージ**
    
- 自分の弱い領域（UI、分析、最適化、税務など）を**外注ではなくスキルで補う**発想
    
- 生成したものを鵜呑みにせず、**検証・改善・比較**（とくに数値）まで回すと強い
