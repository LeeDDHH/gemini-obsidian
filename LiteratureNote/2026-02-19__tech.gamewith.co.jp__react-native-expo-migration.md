# iOS・Android 네이티브 앱을 React Native + Expo로 마이그레이션하기 최적의 시점

**출처**: [GameWith Developer Blog](https://tech.gamewith.co.jp/entry/2026/02/18/113000)
**저자**: @peka3
**발행일**: 2026-02-18
**도메인**: tech.gamewith.co.jp

## TL;DR

- GameWith가 Monster 공략 앱을 네이티브에서 React Native + Expo로 성공적으로 전환한 사례
- 네이티브 전담 인원 감소 & React 개발자 증가 → 팀 역량 활용 극대화
- AI 에이전트 시대에 맞는 CLI 자동화 친화적 환경 구축
- 기존 수개월 걸리던 작업이 AI 에이전트 활용으로 수시간으로 단축

## 요약

GameWith 엔지니어가 Monster 공략 앱을 네이티브(iOS/Android)에서 React Native + Expo로 마이그레이션한 경험을 공유합니다.

### 마이그레이션 이유

- 네이티브 전담 팀원 감소 반면 React 개발자 증가
- AI 에이전트 시대에 대비한 빠른 PDCA 사이클 구축 필요

### Expo를 선택한 이유

- **CLI 기반 완전 자동화**: AI 에이전트와 최적 호환 (GUI 불필요)
- **네이티브 컴포넌트 사용**: OS 업데이트 추적 용이
- **비용 효율**: 사용량 기반 과금 구조
- **Fast Refresh**: 개발 피드백 루프 단축

### 주요 고려사항

- 광고 SDK 통합이 주요 과제
- Firebase와 FCM은 기존 인프라 그대로 활용 가능
- 레거시 Xcode 개발 환경에서 벗어나 현대적 개발 경험으로 전환

### 결과

AI 에이전트를 적극 활용하여 이전에 수개월 걸리던 작업이 수시간으로 단축되었음.

## 핵심 포인트

- AI 에이전트 친화적 환경 = CLI 자동화 + 빠른 피드백 루프
- 팀의 기술 스택 변화(React 비중 증가)를 기술 전환의 트리거로 삼은 사례
- Expo는 AI 에이전트 활용 개발 워크플로우와 궁합이 좋음

## 내 적용 아이디어

- 모바일 앱이 있을 경우 네이티브 → RN+Expo 전환 타이밍 판단 기준으로 참고
- AI 에이전트 개발 환경 구축 시 CLI 친화성을 주요 지표로 고려
