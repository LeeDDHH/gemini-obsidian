## TL;DR

- `git worktree` の利便性を高めるCLIツール `git-wt` の導入記録
- ワークツリーを `.worktrees/` ディレクトリに統一配置することで管理しやすくなった
- ghqユーザー向けの解決策として特に有効
- goでインストール可能、zshとpecoとの組み合わせで快適な操作が実現
- 並行作業（PR並列対応など）に git worktree は強力

## 要約

著者（koic）が `git-wt` を導入した経緯と設定方法についてのメモ記事（2026年1月23日公開）。

**課題**: git worktreeは並行作業に便利だが、ワークツリーのディレクトリ配置が問題。ghqでリポジトリを管理していると、worktreeをどこに置くか迷い、管理しづらくなる。

**解決**: `git-wt` ツール（songmuさんが機能追加）を使うことで、ワークツリーを全てリポジトリ内の `.worktrees/` ディレクトリに統一配置できるようになった。

## セットアップ手順

1. **インストール**: `go install github.com/k1LoW/git-wt@latest`
2. **gitconfig設定**: ベースディレクトリを `.worktrees` に設定
3. **zshrc**: `eval "$(git wt --init zsh)"` を追加
4. **peco連携**: フィルタリング機能でworktree一覧から素早く選択

## 核心ポイント

- **配置の統一**: `.worktrees/` への集約で「どこにある？」問題が解消
- **ghq互換性**: ghq管理のリポジトリでも問題なく使える
- **実用性**: 「顧客が本当に求めていたもの」と著者が絶賛するほど使い勝手が向上

## 内 적용 아이디어

- 複数PRの並行作業時に git worktree + git-wt は有効
- zsh + peco の組み合わせでのワークツリー切り替えは快適そう
- Rubocopなど時間のかかるCIを複数並行で対応するのに最適
