## 요약

utterances는 GitHub 이슈를 기반으로 구축된 경량 댓글 위젯입니다. 블로그나 위키 페이지 등에 GitHub 이슈를 활용하여 댓글 기능을 추가할 수 있습니다. 이 서비스는 오픈 소스이며, 사용자 추적이나 광고가 없고 항상 무료로 제공됩니다. 모든 데이터는 GitHub 이슈에 저장되므로 특정 서비스에 종속될 위험이 없습니다. GitHub의 CSS 툴킷인 Primer로 스타일링되어 있으며, 다크 테마를 지원하고 가볍게 작동합니다.

작동 방식은 utterances가 로드될 때 GitHub 이슈 검색 API를 사용하여 페이지 URL, 경로 또는 제목을 기반으로 해당 페이지와 연결된 이슈를 찾습니다. 일치하는 이슈가 없으면 첫 댓글 작성 시 utterances-bot이 자동으로 이슈를 생성합니다. 사용자는 GitHub OAuth를 통해 utterances 앱에 권한을 부여하여 댓글을 작성하거나, GitHub 이슈에 직접 댓글을 달 수 있습니다.

설정은 사용할 GitHub 저장소를 지정하고, 블로그 게시물과 GitHub 이슈 간의 매핑 방식(예: 페이지 경로, URL, 제목 등)을 선택합니다. 또한, utterances가 생성하는 이슈에 할당할 라벨과 댓글 위젯의 테마를 선택할 수 있습니다. 마지막으로 제공되는 스크립트 태그를 블로그 템플릿에 추가하면 댓글 기능이 활성화됩니다.

## 핵심 포인트 / 인용
*   **핵심 주장**: utterances는 GitHub 이슈를 활용하여 블로그나 웹사이트에 간편하고 프라이버시 친화적인 댓글 기능을 제공하는 오픈 소스 위젯입니다.
*   **근거**:
    *   GitHub 이슈를 백엔드로 사용하므로 별도의 데이터베이스나 관리 시스템이 필요 없습니다.
    *   사용자 추적, 광고, 비용이 없어 개인 블로그나 오픈 소스 프로젝트에 적합합니다.
    *   간단한 설정과 스크립트 삽입으로 쉽게 통합할 수 있습니다.
*   **리스크**:
    *   GitHub 서비스 의존성이 있습니다. GitHub에 문제가 발생하면 댓글 기능도 영향을 받습니다.
    *   댓글 기능 외의 고급 기능(예: 이미지 업로드, 투표 등)은 제한적입니다.
    *   GitHub 계정이 없는 사용자는 댓글 작성이 어렵습니다.
*   **적용 포인트**:
    *   개발 블로그, 기술 문서 사이트, 개인 포트폴리오 등 GitHub 사용자가 주 타겟인 웹사이트에 댓글 기능을 추가하기에 이상적입니다.
    *   개인 정보 보호를 중요하게 생각하며, 서버리스 방식으로 댓글 기능을 구현하고자 할 때 유용합니다.
