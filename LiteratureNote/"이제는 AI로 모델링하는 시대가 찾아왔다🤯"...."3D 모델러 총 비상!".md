# "이제는 AI로 모델링하는 시대가 찾아왔다🤯"...."3D 모델러 총 비상!"
https://www.youtube.com/watch?v=fa5oGTUYH5M

## **핵심 요약**

- 영상은 **“AI는 이제 렌더링을 넘어 ‘모델링(형상 생성/편집)’을 텍스트로 수행하는 시대”**가 왔다고 주장한다.
    
- 소개하는 방식은 **Rhino(라이노)에서 MCP 플러그인으로 ‘로컬 PC 작업’을 AI가 수행**하게 만들고, **Claude Desktop**을 연결해 텍스트 명령으로 모델링을 자동화하는 흐름이다.
    
- 설치는 다소 복잡하지만, 핵심은 **MCP 설치 → Claude Desktop 설치 → 설정 파일(JSON) 교체/붙여넣기 → 클라우드 프로세스 완전 종료 → Rhino에서 MCP 서버 구동 → Claude 재실행으로 연결** 순서로 진행된다.
    

---

## **설치/연결 흐름(영상 내용 기반)**

1. Rhino(또는 플러그인 관리자)에서 **MCP** 검색 후 설치
    
2. 플러그인 안내 페이지(스텝 바이 스텝) 따라 **필수 구성요소 설치(윈도우 인스톨 등)**
    
3. **Claude Desktop 설치 및 로그인**
    
4. Claude Desktop에서
    
    - 설정(Settings) → 개발자(Developer) → 구성 편집(Edit config)
        
    - 해당 설정을 **메모장으로 열어** 안내된 **중괄호(JSON 블록) 전체를 복사/붙여넣기 후 저장**
        
    
5. **Claude 완전 종료** + 작업 관리자에서 **Claude 관련 프로세스까지 종료**
    
6. Rhino 재실행 → MCP 메뉴에서 **서버 시작(호스트/포트 표시)** 확인
    
7. Claude Desktop 재실행 → **Rhino-MCP 연결 완료**

## **데모로 보여준 “텍스트 모델링” 예시 7가지**

- **계단 모델링**: 박스 내부 계단 생성 → 높이(예: 175mm) 변경 → 계단참 포함 회전 계단으로 변형
    
- **그리드 기둥 배치**: 선택된 기둥을 그리드 교차점에 자동 배열(정중앙 정렬 조건 포함)
    
- **루버 생성**: 새 레이어 생성 후 유리창 주변 수직 루버 배치 → 조건(전체 둘레, 모서리 필수, 깊이 500mm 등)로 재생성
    
- **난간 생성/수정**: 커브 기반 난간 생성(높이 1100) → 간격 조정/수평 부재 추가 → 원형이 아닌 바 형태로 수정
    
- **선 기준 높낮이 조정(타워/박스 변형)**: 커브에 가까울수록 낮게, 멀수록 높게 등 “규칙 기반” 변형 시도(박스 수 많으면 실패 가능)
    
- **랜덤 웨이브 타워**: 원통 내부에 비틀림/웨이브 형태의 랜덤 빌딩 생성 시도
    
- **패턴/매핑(벽돌 패턴)**: 곡면 UV 좌표 기반 벽돌 패턴 배치 시도(층간 엇갈림 조건 등 디테일 수정은 한계 존재)

## **한계와 인사이트(영상 결론)**

- 결과는 “초창기 이미지 생성기” 같은 느낌으로 **잘 되는 것/안 되는 것의 편차**가 있다.
    
- **모델 수(오브젝트 수)**가 많거나 조건이 복잡하면 실패하거나 엉뚱한 결과가 나올 수 있다.
    
- 그럼에도 **Grasshopper(그레스호퍼)로 코딩해야 할 작업을 텍스트로 빠르게 시도**할 수 있어 생산성 잠재력이 크다.
    
- 영상은 **Rhino + Grasshopper 이해도가 있으면 AI 모델링 도구를 더 정확히 테스트/활용**할 수 있다고 강조하며, 해외 탑티어 건축사무소가 Rhino/Grasshopper 역량을 중시한다는 맥락으로 마무리한다.
