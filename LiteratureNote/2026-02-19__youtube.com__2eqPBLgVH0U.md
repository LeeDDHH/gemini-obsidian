# ☆클로드 스킬, 서브에이전트, 커맨드 개념 몰랐던 사람 한번에 이해시켜드림

**채널**: 코딩알려주는누나
**길이**: 25:35 | **업로드**: 2026-02-09
**출처**: https://www.youtube.com/watch?v=2eqPBLgVH0U

## TL;DR

- 클로드 코드를 단순한 채팅 도구로만 쓰는 건 AI 기능의 10%밖에 활용하지 못하는 것이다.
- 개발자는 CEO 마인드셋으로 전환하여, AI 서브에이전트를 기획자·개발자·테스터 역할로 분리해 운용해야 한다.
- 서브에이전트를 사용하면 결과물의 일관성과 품질이 높아지고 생산성이 크게 향상된다.
- 에이전트 없이 단순 요청만 하면 결과물이 매번 달라지는 블랙박스 문제가 발생한다.
- CLAUDE.md, 슬래쉬 커맨드, 스킬 파일을 통해 AI에게 구체적인 역할과 지침을 주는 것이 핵심이다.

## 요약

영상은 클로드 코드를 단순히 채팅창에서 코드 구현만 요청하는 방식이 얼마나 비효율적인지 문제 제기로 시작한다. 진행자는 AI로 코드를 짜면서도 설계·테스트·코드리뷰를 모두 사람이 직접 하는 것이 "구시대적 발상"이라고 지적한다.

핵심 메시지는 **CEO 마인드셋**으로의 전환이다. 회사에서 CEO가 "올해 중고래 플랫폼 만들어봐"라고 단순하게 지시하면, 기획자·개발자·QA 테스터들이 각각 역할을 나눠 처리한다. 마찬가지로 개발자도 AI 서브에이전트들을 각 전문 역할로 만들어 두고, 자신은 고수준의 지시만 내리면 된다는 것이다.

**에이전트 없이 쓸 때의 문제점**: "투두 앱 만들어줘"라고 두 번 요청했을 때 결과물이 매번 다르게 나온다. 한 번은 탭이 있는 앱, 다음에는 탭이 없는 앱. AI가 어떤 사고로 결과를 만들었는지 알 수 없는 블랙박스 상태가 된다.

**서브에이전트의 역할**: 에이전트는 구체화된 요구사항과 계획을 가지고 특정 역할에 최적화된 서브에이전트를 호출한다. 서브에이전트들이 기획→개발→테스트까지 전체 사이클을 처리하고 결과를 보고한다.

**실습 내용**: 클로드 코드에서 `/` 슬래쉬 명령어를 눌러 "Create New Agent"를 선택하고, 에이전트의 역할·요구사항·동작 방식을 기술하는 방법을 보여준다. 에이전트는 Personal(모든 프로젝트에 적용) 또는 Project(해당 프로젝트에만 적용) 레벨로 생성 가능하다. AI와 함께 에이전트 스펙을 작성하면 손쉽게 만들 수 있다.

**CLAUDE.md와 커맨드**: CLAUDE.md 파일에 프로젝트 규칙·컨텍스트를 미리 작성해 두면 AI가 자동으로 읽고 일관된 행동을 한다. 슬래쉬 커맨드(`.claude/commands/`)를 사용하면 반복 작업을 자동화할 수 있다.

결론적으로 클로드 코드의 真 가치는 서브에이전트 시스템에 있으며, 이를 제대로 활용하면 기획부터 배포까지 전 사이클을 자동화하여 생산성을 비약적으로 높일 수 있다.

## 핵심 포인트

- **CEO 마인드셋 전환**: 직접 다 하는 직원에서 서브에이전트를 부리는 CEO로 역할을 바꿔야 한다.
- **일관성 문제 해결**: 에이전트 없이는 같은 요청에도 결과가 매번 다른 블랙박스가 된다.
- **서브에이전트 생성법**: 슬래쉬 커맨드 → Create New Agent → 역할 기술. Personal/Project 레벨 선택 가능.
- **CLAUDE.md 활용**: 프로젝트 규칙·컨텍스트를 미리 작성해 두면 모든 에이전트가 일관된 지침을 따른다.
- **슬래쉬 커맨드**: `.claude/commands/`에 Markdown 파일로 정의하는 자동화 워크플로우.
- **스킬 개념**: AI에게 특정 도구나 능력을 부여하는 개념으로, 서브에이전트의 역량을 확장한다.
- **생산성 혁신**: 기획→개발→테스트→배포 전 사이클을 AI 에이전트가 담당, 사람은 방향만 제시.
