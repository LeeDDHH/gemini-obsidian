이 GitHub Docs 튜토리얼은 GitHub Copilot을 활용하여 새로운 프로그래밍 언어를 학습하는 방법을 안내한다. 이미 프로그래밍 경험이 있는 개발자가 새로운 언어를 효율적으로 익히는 데 초점을 맞추고 있다.

주요 내용은 다음과 같다:
*   **기본 학습:** Copilot Chat을 이용해 새로운 언어의 기본적인 개념과 기존에 알던 언어와의 차이점을 탐색한다. 언어의 최적 사용처, 다른 언어와의 주요 차이점, 설치 방법, 에러 처리 메커니즘, 의존성 및 패키지 관리, 중요한 라이브러리 및 프레임워크, 테스트 프레임워크, 초보자가 흔히 저지르는 실수 등을 질문할 수 있다.
*   **개인 트레이너로서의 Copilot 활용:** Copilot에게 많은 코드를 작성하게 하는 대신, 제안된 코드에 대한 설명을 요청하며 코드의 각 부분을 완전히 이해할 때까지 질문을 반복하는 방식을 권장한다. 이해하지 못한 코드는 사용하지 않도록 경고한다.
*   **간단한 프로그램 작성:** 먼저 알고 있는 언어로 쉽게 작성할 수 있는 간단한 프로그램을 새로운 언어로 만들어보며 Copilot의 도움을 받는다. 이때 Copilot에게 코드에 대한 설명을 요청하고 스스로 프로그램을 확장해 나가는 것이 중요하다.
*   **주석 및 인라인 제안 활용:** IDE에서 주석을 작성하여 Copilot에게 새로운 프로그램의 기본 코드를 제안하게 하거나, 인라인 제안을 통해 코드를 완성하는 방법을 소개한다.
*   **구체적인 질문:** 새로운 언어의 구문과 관용구에 익숙해지기 위해 Copilot에게 특정 작업을 수행하는 작은 코드 단위에 대해 명확하고 범위가 좁은 질문을 한다.
*   **기존 코드 변환:** 이미 익숙한 언어로 작성된 자율적인 코드(예: 함수)를 새로운 언어로 변환하여, 같은 작업이 새로운 언어에서 어떻게 구현되는지 비교 분석하며 학습한다.
*   **기존 코드 읽기:** 새로운 언어로 작성된 기존 코드를 읽으며 Copilot Chat을 통해 파일의 목적과 코드의 흐름을 이해하고, 개선 방안을 문의한다.
*   **Copilot의 한계 인식:** Copilot은 유용한 도구이지만 완벽하지 않으므로, 항상 제안된 코드를 확인하고 이해한 후 사용해야 한다. 오류가 발생하면 Copilot에게 수정 요청을 하고, 코딩 스타일 가이드라인 준수를 위해 리포지토리 사용자 정의 지침을 활용하는 방법도 언급한다.
