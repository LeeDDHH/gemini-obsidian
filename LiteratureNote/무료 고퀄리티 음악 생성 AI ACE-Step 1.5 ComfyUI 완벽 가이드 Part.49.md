## **요약: ComfyUI로 무료 음악 생성(ACE-Step 1.5) 워크플로우 정리**

  

### **1) 왜 ACE-Step 1.5인가? (핵심 장점 4가지)**

- **로컬 구동 + 낮은 사양 대응**: 로컬에서 AI 모델을 돌리며, **RTX 3060 4GB VRAM**에서도 동작한다고 소개됨.
    
- **“완성된 곡”을 생성**: 다른 모델은 길이 설정 시 중간에 끊기는 경우가 있는데, ACE-Step은 **처음부터 끝까지 완성된 형태의 곡**을 만들어줌.
    
- **세부 제어 가능**: 스타일뿐 아니라 **BPM 등 구체 파라미터까지 제어**할 수 있음.
    
- **다국어 지원**: **50개 이상의 언어 지원**(발음/딕션은 추가 검증 필요하다고 언급).
    

---

### **2) 모델 다운로드 구조(허깅페이스)와 선택 전략**

  

모델 페이지에 2가지 형태가 있음:

- **Checkpoint 모델(통합형)**
    
    - 하나의 모델에 **CLIP + VAE 포함**
        
    - 용량이 큼(약 10GB 언급)
        
    
- **Split 모델(분리형)** ✅ 이번 영상에서 사용
    
    - **Diffusion 모델 / Text Encoder / VAE**가 분리되어 있음
        
    - Diffusion에 **Turbo 모델**이 있어 **생성 속도 목적**으로 Split을 선택
        
    

---

### **3) 모델 저장 위치(ComfyUI 폴더)**

- **Diffusion 모델** → models/diffusion_models/ (터보 모델 사용)
    
- **Text Encoder 모델(3개 중 선택/조합 가능)**
    
    - 이번 영상: **0.6B + 1.7B**를 조합 사용
        
        → models/text_encoders/
        
    
- **VAE 모델(1개)** → models/vae/
    

---

### **4) 워크플로우 구성 포인트**

  

#### **(1) 모델 로딩 그룹**

- Diffusion: **ACE-Step 1.5 Turbo**
    
- Text Encoders: **0.6B + 1.7B**
    
- Type: **ACE**
    
- VAE: **ACE-Step VAE**
    

  

#### **(2) 음악 길이 설정**

- 길이: **120초(2분)**
    
- 품질 권장 범위: **90~120**에서 가장 좋다고 소개
    

  

#### **(3) 프롬프트 입력 방식(많이 막히는 부분 해결)**

- 프롬프트는 2단 구성:
    
    - **상단**: 음악 스타일/요구사항
        
    - **하단**: 가사(lyrics)
        
    
- 해결 팁:
    
    - 워크플로우에 **프롬프트에 쓰기 좋은 키워드**와
        
        **가사 구성용 태그**를 정리해 두고,
        
    - 이를 조합하거나,
        
    - **ChatGPT에 “요구사항+예시”를 주고 프롬프트/가사 생성**을 시키면 됨
        
        → 결과를 복붙하면 빠르게 진행 가능
        
    

---

### **5) 주요 옵션(중요한 것만)**

- **BPM**
    
    - 높을수록: 빠른 댄스/비트 중심
        
    - 낮을수록: 느린 발라드/잔잔한 리듬
        
    - 정확히 모를 때: **ChatGPT에 곡 스타일을 주고 BPM 추천** 받기 → 그대로 입력
        
    
- **Language**
    
    - 가사 언어와 동일하게 설정 권장 (예: 한국어 가사면 한국어)
        
    
- **Sampler**
    
    - Steps: **8**
        
    - (터보 모델이라 낮은 스텝에서도 빠르게 생성 가능)
        
    - Sampler: **Euler**
        
    - Schedule: **Simple**
        
    

---

### **6) 실제 생성 결과(속도/품질)**

- 2분짜리 음악 생성 시간: **약 35초**
    
- 품질: 전반적으로 **수준급으로 잘 생성**
    
- 한국어: **일부 구간 발음/딕션이 약간 미흡**하다고 평가
    

---
