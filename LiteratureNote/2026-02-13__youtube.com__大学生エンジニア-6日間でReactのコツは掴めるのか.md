# 【Day192】大学生エンジニア、6日間でReactのコツは掴めるのか

## TL;DR
프로그래밍 학습 258일차(Day 192) 대학생 엔지니어 이치카가 React 공식 튜토리얼의 틱택토(Tic-Tac-Toe) 응용 편을 라이브 코딩으로 풀어가는 과정을 담은 생방송 기록. JSX 내 조건부 렌더링(삼항 연산자), 이중 루프를 활용한 보드 렌더링 리팩토링, 정렬 토글 버튼 구현, 승리 라인 하이라이트 등 React 공식 문서의 응용 과제 5개를 해결하며, 시행착오와 ChatGPT 활용을 통한 실시간 문제 해결 과정을 보여준다.

## 섹션별 요약

### 1. 전일 복습 (도입부)
- 시청자 인사 및 라이브 환경 세팅
- 어제 학습한 내용 복습: state 변수는 최소화, `map()` 사용 시 미사용 인수에 `_` 표기, state 갱신 시 React 재렌더링 원리, 버튼의 기본 색상 문제, 보드 컴포넌트 구조와 props 전달

### 2. 응용 과제 1: 조건부 렌더링 (삼항 연산자)
- **문제**: 현재 위치(current move)일 때는 버튼 대신 텍스트로 표시
- **시행착오**: JSX 내에서 `{}` 없이 조건 분기를 시도하여 실패 → ChatGPT에 질문하여 삼항 연산자 활용법 학습
- **해결**: `move === currentMove ? <span>텍스트</span> : <button>...</button>` 패턴 적용
- **배움**: JSX 내부에서는 `{}`로 감싸야 JavaScript 표현식 사용 가능

### 3. 응용 과제 2: 이중 루프로 보드 렌더링
- **문제**: 9개의 Square를 하드코딩하지 않고 이중 루프로 렌더링
- **시행착오**: "하드코딩"의 의미 자체를 이해하지 못해 ChatGPT에 확인 → 행(row)·열(col) 배열을 만들어 `map()`으로 반복 렌더링
- **해결**: `[0,1,2].map(row => ...)` 외부 루프와 `[0,1,2].map(col => ...)` 내부 루프로 `row*3+col` 인덱스 계산
- **배움**: `key` prop의 중요성, 이중 루프에서의 인덱스 계산법

### 4. 응용 과제 3: 정렬 토글 버튼
- **문제**: 이동 목록을 오름차순/내림차순으로 토글하는 버튼 추가
- **시행착오**: `reverse()` 함수가 원본 배열을 변경(mutate)하여 React 상태 관리와 충돌 → 스프레드 연산자(`[...array].reverse()`)로 복사 후 역정렬
- **해결**: `isAscending` state 추가, 조건에 따라 moves 배열을 복사 후 역순 정렬
- **배움**: React에서 배열 불변성(immutability)의 중요성, `reverse()`가 원본을 mutate한다는 점

### 5. 응용 과제 4: 행·열 좌표 표시
- **문제**: 각 이동에 대해 (행, 열) 좌표를 표시
- **해결**: 인덱스를 `Math.floor(index/3)+1`(행)과 `(index%3)+1`(열)로 변환
- **배움**: 1차원 인덱스에서 2차원 좌표 변환 공식

### 6. 응용 과제 5: 승리 라인 하이라이트
- **문제**: 승자 결정 시 해당 3개 칸을 하이라이트, 무승부 시 별도 표시
- **시행착오**: `calculateWinner` 함수에서 승리 라인을 반환하도록 수정 시도 → 기존 코드 구조와 호환 문제
- **해결**: `calculateWinner`의 반환값을 `{winner, line}` 객체로 변경, 승리 칸에 CSS 클래스 추가, 모든 칸이 채워지고 승자가 없으면 무승부 표시
- **배움**: 함수 반환값 구조 변경 시 전체 코드에 미치는 영향, CSS를 통한 동적 스타일링

## 핵심 주장/근거/예시
- **주장**: 프로그래밍 초보자도 공식 문서의 단계적 과제를 따라가며 시행착오를 겪으면 6일 내에 React의 핵심 개념(state, props, 조건부 렌더링, 리스트 렌더링, 불변성)을 체득할 수 있다.
- **근거**: 라이브 코딩에서 실시간으로 에러를 만나고 ChatGPT와 시청자 도움을 받아 해결하는 과정 자체가 학습 효과를 보여줌.
- **리스크**: 자동 번역 자막이므로 일본어 원어의 미묘한 뉘앙스가 일부 손실됨. 라이브 특성상 잡담과 반복이 많아 핵심 내용 밀도가 낮음.

## 실무 적용 포인트
- React 학습 시 공식 튜토리얼의 "응용 과제"까지 반드시 시도할 것 — 기본 과제만으로는 핵심 패턴을 놓치기 쉬움
- `reverse()`, `sort()` 등 mutate 메서드 사용 시 반드시 스프레드 연산자로 복사 후 사용 (React 불변성 원칙)
- 라이브 코딩/학습 기록을 콘텐츠로 만드는 방식 자체가 학습 동기 부여와 커뮤니티 형성에 효과적
