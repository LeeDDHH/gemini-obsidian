# Codex 는 컴포넌트 재사용을 잘한다?
https://www.youtube.com/watch?v=BqMfyQSv3xE

## **요약**

- **관찰된 차이점**
    
    - **코덱스(Codex)**: 레포(코드 전체) 맥락을 먼저 파악한 뒤, 이미 존재하는 **디자인 시스템 컴포넌트를 찾아 재사용**하는 경향이 있다.
        
    - **클로드(Claude)**: 동일한 요구(“디자인 룰스 참고해서 가독성 있고 예쁘게 구현”)를 줬을 때, **기존 컴포넌트를 재사용하지 않고 새로 직접 구현**하는 결과가 나왔다.
        
    
- **핵심 원인(프롬프트 관점)**
    
    - 질문/지시문에서 **“컴포넌트 재사용”을 명시하지 않으면** 클로드는 재사용을 덜 하는 경향이 있다.
        
    - 반대로, **“기존 디자인 시스템 컴포넌트를 우선 재사용해줘”** 같은 문장을 넣으면 클로드도 재사용을 잘 할 가능성이 높다.
        
    - 즉, “클로드는 항상 직접 구현한다”가 아니라, **재사용을 유도하는 명시적 프롬프트가 필요**하다는 이야기.
        
    
- **실무 적용 포인트**
    
    - 클로드 코드를 쓸 때는 요구사항에 기능/스타일뿐 아니라,
        
        - “가능하면 기존 디자인 시스템 컴포넌트를 찾아 재사용”
            
        - “새 구현이 필요하면 그 이유를 설명”
            
        - “대체 컴포넌트가 있으면 우선 사용”
            
            같은 **재사용 지시를 프롬프트에 포함**하면 결과 일관성이 좋아진다.
            
        
    

  

## **키워드 (최대 3개)**

- 디자인 시스템 재사용
    
- 프롬프트 명시성
    
- 코드베이스 컨텍스트 이해
