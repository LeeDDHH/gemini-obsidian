## **진행 흐름**

  

### **1) 전날 내용 복습**

- **state(상태)는 가능한 적게 유지**하는 게 버그를 줄인다는 포인트를 복습
    
- map에서 **인덱스만 쓰더라도 첫 번째 인자(요소)를 자리상 둬야 함**(또는 _로 생략)
    
- React가 “쉬운” 이유: **state가 바뀌면 자동으로 리렌더링**되기 때문
    
- 컴포넌트 구조에서 Game이 상위에서 상태를 관리하고, Board로 내려보내는 이유(상태 소유 위치)
    

---

### **2) “현재 위치만 버튼 대신 텍스트로 표시” 문제 해결**

- 하고 싶었던 것: **현재 선택된 히스토리(move)만 버튼이 아니라 텍스트(span)로 표시**
    
- 막힌 이유:
    
    - JSX 안에서 조건 분기를 쓸 때 **그대로** **if****를 쓰기 어렵고**, 보통 **삼항 연산자(조건 ? A : B)** 또는 논리 연산으로 처리
        
    - JSX에서 JS식을 쓸 때 **{ ... }****를 깜빡해서 문장이 그대로 렌더링되는 문제**가 발생
        
    
- 해결:
    
    - {조건 ? <span>...</span> : <button>...</button>} 형태로 분기
        
    - “JSX 안에서 JS는 {}로 감싼다”를 다시 체득
        
    

---

### **3) “9칸을 하드코딩하지 말고 2중 루프로 렌더링” 문제 해결**

- 과제 의도:
    
    - Square 9개를 **손으로 나열(하드코딩)**하지 말고,
        
    - **행(3) × 열(3) = 9칸을 2중 루프(map)로 생성**하라
        
    
- 핵심 난관:
    
    - (row, col)에서 **각 칸의 index를 어떻게 계산하느냐**
        
    
- 해결 공식:
    
    - **index = row * 3 + col**
        
    
- 추가로 해결한 부분:
    
    - map으로 만든 요소에는 **key****가 필요** (행/열 혹은 계산한 index를 key로 사용)
        
    - 괄호/중괄호 짝이 꼬여 “빨간 화면(문법 에러)”이 나오는 문제를 **정리/포맷팅하며 해결**
        
    - 중복으로 렌더링되는 줄(기존 하드코딩 부분)을 제거해서 **18칸이 되지 않게** 정리
        
    

---

### **4) “히스토리 표시 순서를 토글(정방향/역방향)하는 버튼” 시도**

- 과제 해석:
    
    - 히스토리 목록(“Go to move #…”)을 **오름차순/내림차순으로 전환하는 버튼** 추가
        
    
- 처음 시도에서 발생한 착각:
    
    - ol reversed 속성만으로 **번호는 바뀌지만**, 실제 데이터(문구/내용) 순서가 기대처럼 안 바뀌는 느낌을 받음
        
    
- 방향 전환:
    
    - 표시만 뒤집는 게 아니라, **렌더링에 사용하는 배열 자체를 역순으로 만들 필요**를 인식
        
    - JS에서 배열을 뒤집는 방법을 탐색:
        
        - 원본 변경이 싫으면 toReversed() 같은 **비파괴(immutable) 방식**을 고려(환경에 따라 지원 여부 차이 있을 수 있음)
            
        
    
- 토글 구현 방식:
    
    - useState로 boolean 상태를 만들고, 버튼 클릭 시 값을 반전
        
    - 상태에 따라 **정방향 배열 vs 역방향 배열**을 선택해서 map에 넣는 구조를 떠올림

## **핵심 학습 포인트 정리**

- JSX에서 조건 분기: **삼항 연산자 / 논리 연산 +** **{}**
    
- map 렌더링: **key 필수**
    
- 2중 루프 렌더링: **row/col로 index 만들기**
    
    - index = row * 3 + col
        
    
- “표시 순서 토글”은 **번호만 뒤집는 것 vs 데이터 배열을 뒤집는 것**이 다름
    
- map 콜백이 **식(표현식) 바디면 return 생략 가능**, 블록 바디면 return 필요
