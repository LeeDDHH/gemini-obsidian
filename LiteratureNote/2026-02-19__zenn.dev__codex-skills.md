## TL;DR

- Codex CLIにSkills機能が実装：エージェントが動的に発見・ロードできるカスタムコマンドパッケージ
- YAML フロントマター＋本文の2段構造で、必要時のみロードされる「段階的開示」設計
- `~/.codex/skills/*/SKILL.md` または `.codex/skills/`（プロジェクト固有）に配置
- AGENTS.mdは常時読込だがコンテキスト消費大→特定状況のみ必要な知識はSkillsに切り出す

## 要約

著者 akihiro（aki_think）がCodex CLIのSkills機能を解説した記事（2025年12月6日）。

**Skillsの基本概念**:
- エージェントが「動的に発見・ロード」できる指示・スクリプト・リソースのパッケージ
- LLMが状況に応じて必要なSkillを自動的に読み込む
- 起動時はスキルの名前と説明のみがコンテキストに含まれ、必要時に本文がロードされる

**なぜ重要か**: AGENTS.mdに全情報を詰め込むとコンテキスト消費が大きくなる問題を解決。「常に必要な情報」と「特定状況のみ必要な情報」を分離できる。

**ファイル構造**:
```
~/.codex/skills/{skill-name}/SKILL.md  # グローバル
.codex/skills/{skill-name}/SKILL.md   # プロジェクト固有（v0.66.0+）
```

**実装例**: error-analyzer スキル（エラー調査と日本語レポート作成の自動化）

## 核心ポイント

- **段階的開示（Progressive Disclosure）設計**: 全情報を事前ロードせず、必要時にオンデマンドで読み込む
- **プロジェクト固有対応**: v0.66.0以降でリポジトリ配下に配置可能（このリポジトリの `.claude/commands/` と同じ発想）
- **コンテキスト効率化**: AGENTS.md vs Skills の分担でトークン消費を最適化

## 内 적용 아이디어

- Claude Codeの `.claude/commands/` と同様の発想。Claude Code のスキルシステムと比較すると理解が深まる
- 大きな CLAUDE.md を持つプロジェクトでも、頻度低いコマンドはSKILL.md化できる
- 「段階的開示」の設計思想は他のAIツール設計にも応用可能
