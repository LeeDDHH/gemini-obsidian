# 추이적 의존관계를 알기 쉽게 보자

**출처**: [Qiita](https://qiita.com/mi_iroha/items/763ae60b04965002d417)
**저자**: mi_iroha (どすこい塾)
**발행일**: 2024-02-22
**도메인**: qiita.com

## TL;DR

- 라이브러리 업데이트 시 예상치 못한 추이적 의존성 변화가 발생할 수 있음
- `dependency-diff-tldr` 도구로 업데이트 전후 의존성 변화를 시각화 가능
- Android Gradle 프로젝트에서 PR 검토 시 투명한 의존성 정보 제공 방법

## 요약

### 추이적 의존관계란?

라이브러리가 다른 라이브러리를 사용할 때 발생하는 간접 의존관계입니다. 예를 들어:
- 라이브러리 A가 라이브러리 B v1.0.0을 사용 중
- A를 새 버전으로 업데이트하면 A가 B v1.2.0을 요구할 수 있음
- 앱에는 자동으로 B v1.2.0이 적용됨 (직접 의존하지 않더라도)

이 변화가 예기치 않은 호환성 문제를 유발할 수 있음.

### 해결 도구: dependency-diff-tldr

- GitHub Releases에서 `dependency-diff-tldr-r8.jar` 다운로드
- 업데이트 전후 의존성을 비교하여 "새로운 의존성" / "업그레이드된 의존성" 목록 출력

### 사용 방법

```bash
# 업데이트 전 의존성 트리 저장
./gradlew :app:dependencies --configuration releaseRuntimeClasspath > old.txt

# 업데이트 후 동일 명령
./gradlew :app:dependencies --configuration releaseRuntimeClasspath > new.txt

# 비교
java -jar dependency-diff-tldr-r8.jar old.txt new.txt
```

## 핵심 포인트

- Android/Gradle 프로젝트에서 라이브러리 업데이트 PR 리뷰 시 의존성 변화를 명시적으로 확인하는 것이 중요
- 추이적 의존관계 변화는 테스트 없이 발견하기 어려움

## 내 적용 아이디어

- Android 프로젝트가 있다면 PR 체크리스트에 의존성 diff 확인 항목 추가
- 패키지 매니저 업데이트 시 항상 추이적 의존성 변화를 명시적으로 검토
