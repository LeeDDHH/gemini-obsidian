# 클로드 스킬, 서브에이전트, 커맨드 개념 몰랐던 사람 한번에 이해시켜드림
https://www.youtube.com/watch?v=2eqPBLgVH0U

## **요약 (핵심 메시지)**

- **단순 채팅으로만 Claude Code를 쓰면 AI를 10%밖에 못 쓴다**는 주장입니다.
    
    많은 사람이 _설계(요구사항 정리/DB 설계)_ → _구현_ → _테스트/리뷰_ 를 결국 “본인이 다” 하고 있어서 생산성이 크게 안 오른다고 말합니다.
    
- 해결책은 **“CEO 마인드셋” + “서브 에이전트(Sub-agent)”** 입니다.
    
    사용자는 사장처럼 “뭘 만들지”만 말하고, **기획자/개발자/테스터 역할을 각각 서브 에이전트로 분리**해서 자동으로 협업하게 만들면,
    
    - 결과물의 **일관성**
        
    - 과정의 **투명성(문서화)**
        
    - 테스트/검증 자동화
        
        가 가능해진다고 설명합니다.
        
    

---

## **왜 결과가 들쭉날쭉한가?**

- 같은 “투두 앱 만들어줘”를 두 번 시키면
    
    - 첫 번째는 탭이 있고
        
    - 두 번째는 탭이 없고
        
    - 질문 흐름도(스택/저장 방식 등) 매번 달라짐
        
        → **AI가 알아서 결정하는 영역이 커져서 “블랙박스”가 된다**고 주장합니다.
        
    

---

## **해결 전략 1: 서브 에이전트로 역할 분리**

  

### **1) 플랜(기획/요구사항 구체화) 에이전트 만들기**

- 목적: “요구사항 구체화 + 기술스택/폴더구조/DB 스키마까지 계획 문서화”
    
- 장점: 사용자가 프롬프트로 장황하게 쓰지 않아도 **계획 문서가 먼저 생성**되어 컨트롤이 쉬워짐
    

  

### **2) 구현(코딩) 에이전트 만들기**

- 목적: “시니어 개발자 관점 + 확장성/클린 아키텍처 기반 구현”
    
- 장점: 작은 앱이라도 역할 분리된 파일 구조/품질이 더 좋아짐
    

  

✅ 핵심 포인트:

**요청 시, 에이전트가 description(설명)을 보고 상황에 맞는 서브 에이전트를 선택/호출**하게 만들어서, 사람 팀처럼 굴린다는 접근입니다.

---

## **해결 전략 2: Skills(규칙/온보딩 문서)로 품질 고정**

- **Skills = 범용 규칙 세트** (온보딩 문서처럼)
    
    - 예: “비전공자도 이해할 수 있게 주석을 한국어로 쉽게 달아줘”
        
    
- 에이전트(역할)와 스킬(규칙)을 분리하면
    
    - 에이전트는 “무슨 일을 하는가”
        
    - 스킬은 “어떤 스타일/룰로 일하는가”
        
        로 정리되어 유지보수가 쉬워짐
        
    

---

## **해결 전략 3: 커맨드화(/develop)로 실행 비용 0에 가깝게 만들기**

- 매번 “서브 에이전트 써서… 스킬 참고해서…”를 타이핑하기 싫으면
    
    - 이를 **슬래시 커맨드로 템플릿화**해서
        
    - /develop + 요구사항 한 줄만으로
        
    - “계획 → 구현(스킬 적용)”까지 자동 진행
        
    

---

## **고급 예시: PDCA 워크플로우 에이전트(플랜→디자인→구현→분석)**

- 특정 서브 에이전트(예: vkit 계열)를 사용해 **회사 개발 흐름(PDCA)** 자체를 에이전트로 구현했다고 소개합니다.
    

  

흐름:

1. **Plan**: 기획 문서 생성
    
2. **Design**: 설계 문서 생성(아키텍처/플로우 등)
    
3. **Do**: 설계대로 구현
    
4. **Check/Analyze**: “문서 대비 실제 구현 일치율” 분석(예: 96% 매칭, 누락 UI 지적 등)
    
5. **Act/Iterate**: 낮은 일치율일 때 수정 루프
    

  

✅ 결과적으로 “문서만 들고 가도 동일한 앱 재생성”이 가능해져서

**재현성(일관성)과 통제력**이 생긴다고 강조합니다.

## **결론: 앞으로의 에너지는 어디에 써야 하나?**

- “내가 더 열심히 프롬프트 쓰기”가 아니라
    
- **내 프로젝트에 맞게 서브 에이전트/스킬/워크플로우를 ‘잘 키우는 것’**이 핵심 역량이라고 정리합니다.
