# AI 시대의 개발자 교육 철학: 기본기 vs AI 의존

## TL;DR

일본 홋카이도 대학은 "AI가 못하는 것의 전문가가 되라"고 가르치고, 한 주니어 개발자는 ChatGPT를 멘토 삼아 2달 만에 Next.js 풀스택 앱을 만들었다. 대학생 이치카는 React 공식 튜토리얼의 응용 과제를 라이브 코딩으로 풀며 시행착오 자체가 학습임을 보여주었다. **AI 시대의 개발자 교육은 "AI를 쓰냐 마냐"가 아니라, "AI가 대체할 수 없는 역량(시스템 설계, 이론적 기반, 창의적 판단)을 어떻게 키우느냐"**가 핵심이다. 기본기 없이 AI에 의존하면 디버깅도, 판단도 불가능하다.

## 내 결론 (현재 시점)

교육에서 AI의 역할에 대한 세 가지 입장이 공존한다:

**1. 보수적 입장 (홋카이도 대학)**
- AI는 초보자의 핵심 감각 발달을 방해한다
- AI가 만든 코드의 교묘한 버그를 찾는 데 시간이 더 걸린다
- 20~30년 유효한 이론(C/C++, 알고리즘, 컴퓨터 구조)을 먼저 익혀라
- **"AI의 뒷수습을 하기보다는 AI가 못하는 것의 전문가가 되라"**

**2. 적극 활용 입장 (ChatGPT 멘토)**
- "5살 아이에게 설명하듯" 프롬프트를 작성하면 AI가 효과적인 멘토가 됨
- 2달 만에 기본 개념 → 풀스택 Next.js 앱 구축까지 가능
- 핵심은 효과적인 프롬프트 작성법과 자기 수준에 대한 맥락 제공

**3. 시행착오 중심 입장 (대학생 이치카)**
- 공식 문서의 응용 과제를 직접 풀어보는 것이 핵심
- 하드코딩의 의미도 모르는 상태에서 시작해도 시행착오를 통해 체득
- ChatGPT와 시청자의 도움을 받되, 직접 에러를 만나고 해결하는 과정 자체가 학습

나의 판단: **세 입장은 상호보완적이다**. 가장 위험한 것은 AI에만 의존하여 "왜 작동하는지" 모르는 상태로 코딩하는 것. 가장 효과적인 학습은:
1. 기본 이론 먼저 (최소한 자료구조, 알고리즘, 메모리 모델)
2. 직접 시행착오를 겪으면서 코딩
3. 막힐 때 AI를 멘토로 활용 (답을 구하는 게 아니라 설명을 구하는 방식)
4. freeCodeCamp 같은 구조화된 커리큘럼으로 체계적 보충

## 반례 / 제약 / 리스크

- **이론 과잉의 위험**: 이론만 파고들면 실무 감각이 떨어질 수 있음. 일본 대학의 온건한 과제 난이도가 오히려 실무 역량 부족으로 이어질 가능성
- **AI 멘토의 정확성**: ChatGPT가 항상 정확한 정보를 제공하지는 않음. 초보자는 잘못된 정보를 구분하기 어려움
- **생존 편향**: ChatGPT로 2달 만에 성공한 사례가 보편적인지는 확인 불가. 실패한 경우는 글을 쓰지 않음
- **문화적 차이**: 일본의 이론 중심 교육이 한국/미국의 실무 중심 교육보다 반드시 우수하다는 근거는 없음
- **기술 변화 속도**: "20~30년 유효한 이론"이라는 주장도, 양자 컴퓨팅이나 패러다임 전환 시 재검토 필요

## 실무 적용 체크리스트

- [ ] **기본기 점검**: 자료구조, 알고리즘, 메모리 모델을 설명할 수 있는지 자가 진단
- [ ] **AI 멘토 활용법 개선**: AI에게 답을 구하지 말고, "왜?"와 "원리"를 물어보는 프롬프트 습관
- [ ] **공식 문서 응용 과제 도전**: React 등 주요 프레임워크의 공식 튜토리얼 응용 과제까지 직접 풀어보기
- [ ] **`reverse()` 같은 mutate 메서드 주의**: React에서 배열 불변성 원칙 — 항상 스프레드 연산자로 복사 후 조작
- [ ] **freeCodeCamp 인증 도전**: 무료·영구 유효한 인증으로 기본기 체계적 보충
- [ ] **학습 기록 공개**: 라이브 코딩이나 블로그 형태로 학습 과정을 공유하면 동기 부여와 커뮤니티 형성에 효과적
