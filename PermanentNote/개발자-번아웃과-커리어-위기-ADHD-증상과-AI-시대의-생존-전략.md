# 개발자 번아웃과 커리어 위기: ADHD 증상과 AI 시대의 생존 전략

## TL;DR
- 2024 Stack Overflow 조사: **프로그래머의 80%가 직무에 불만족**
- 개발자들이 **ADHD 유사 증상** 경험: 개발 환경 자체가 끊임없는 인터럽트와 도파민 루프 구조
- 주요 원인: 컨텍스트 스위칭 (평균 23분 소요), 즉각적 보상 중독, 번아웃, 직업 종말론
- AI 시대의 반복되는 **"개발자 종말론"**: 노코드 → 자동화 → LLM → AGI
- 생존 전략: 기술 업데이트 ≠ 진정한 성장, **의지보다 환경 설계**, 의사결정/설계 역량 강화

## 내 결론(현재 시점)
**개발자의 불행은 개인 문제가 아니라 구조적 문제다.** 개발 환경은 본질적으로 비선형적이고, 짧은 도파민 루프로 설계되어 있어 장기 집중을 방해한다. AI는 단순 반복 작업을 대체하지만, **의사결정, 설계, 리뷰**는 여전히 인간의 영역이다. 

### 핵심 패러다임 전환
1. **번아웃 원인**: 개인 의지 부족 → 구조적 컨텍스트 스위칭 + 도파민 중독
2. **ADHD 증상**: 선천적 장애 → 개발 환경이 ADHD처럼 행동하도록 훈련
3. **학습 패러다임**: 생존을 위한 학습(Latest Framework) → 성장을 위한 학습(원리, 설계)
4. **AI 위협**: 직업 종말 → 저부가가치 작업 대체, 고급 역량 수요 증가

### 실무 검증 데이터
- **Stack Overflow 2024**: 프로그래머 80% 직무 불만족
- **컨텍스트 스위칭**: 방해 후 재집중 평균 23분
- **도파민 루프**: 버그 수정 → 테스트 통과 → 배포 성공 = 즉각적 보상 구조
- **기술 부채**: Google의 10가지 카테고리 (코드 품질, 테스트, 문서, 의존성 등)

### ADHD 증상의 구조적 원인
- [[ADHD 개발자 경험]]
  - "개발자가 ADHD가 많은가, 아니면 일이 그렇게 만드는가?"
  - **비선형적 워크플로우**: 디버깅 → Slack → 코드 리뷰 → 2시간 리팩터링 (방해 후 재집중 23분)
  - **도파민 트랩**: 버그 수정/테스트 통과/배포 = 즉각적 보상 → SNS/게임 중독 구조와 유사
  - **하이퍼포커스 vs 번아웃**: ADHD 성향 개발자는 복잡한 문제에 몰입하지만 무기력/미루기 쉬움
  - **해결책**: "더 집중해라" ✗ → **환경/워크플로우 설계** ✓ (포모도로, 알림 차단, 짧은 성공 리스트)

### AI 시대의 직업 종말론
- [[AI 시대_ 반복되는 개발자 종말론]]
  - 역사: 노코드 도구 → 자동화 → LLM → AGI
  - 실제: 저부가가치 작업은 대체되지만, 설계/의사결정/리뷰는 인간 영역
  - 개발자 역할 진화: 코드 작성자 → AI 프롬프터 + 의사결정자 + 리뷰어

### 학습 패러다임의 전환
- [[성장을 위한 학습인가, 생존을 위한 학습인가?]]
  - **생존 학습**: 최신 프레임워크 따라잡기 (React 18, Next.js 15, Svelte 5...)
  - **성장 학습**: 원리 이해 (웹 표준, 알고리즘, 아키텍처 설계)
  - AI 시대에는 성장 학습이 더 중요: 원리를 아는 사람이 AI를 잘 활용

### 기술 부채의 체계적 분류
- [[Google이 추출한 10가지 카테고리로 기술 부채를 재정의하다]]
  - 코드 품질, 테스트, 문서화, 의존성, 인프라, 보안, 접근성, 성능, 데이터, 프로세스
  - 개발자 불행의 주요 원인 중 하나: 쌓이는 기술 부채

## 반례/제약/리스크

### 1. 모든 개발자가 불행한 것은 아니다
- 20%는 직무에 만족: 좋은 팀 문화, 합리적 업무량, 성장 기회
- **대응**: 불만족 요인을 개인이 아닌 조직 차원에서 제거

### 2. ADHD 증상 ≠ ADHD 진단
- 개발 환경이 ADHD처럼 행동하게 만들지만, 실제 ADHD는 의학적 진단 필요
- **대응**: 심각한 경우 전문가 상담, 가벼운 경우 워크플로우 개선

### 3. AI 종말론의 과장
- 역사적으로 새 기술은 일자리를 대체하지 않고 재편함
- 저급 코더는 대체되지만 고급 엔지니어 수요는 증가
- **대응**: AI 도구를 활용해 부가가치 높은 작업에 집중

### 4. 학습의 함정
- 성장 학습만 하면 실무에서 뒤처짐 (최신 기술 무시)
- 생존 학습만 하면 표면적 지식 축적 (원리 부재)
- **대응**: 8:2 원칙 (80% 성장 학습, 20% 생존 학습)

### 5. 환경 설계의 한계
- 조직 문화, 팀 프로세스는 개인이 통제 불가능
- **대응**: 통제 가능한 범위(개인 워크플로우, 도구, 학습)에 집중, 최악의 경우 이직

## 실무 적용 체크리스트

### 번아웃 예방 (매일)
- [ ] 포모도로 기법: 25분 집중 + 5분 휴식
- [ ] 알림 차단: Slack/이메일 배치 처리 (1일 3회)
- [ ] 컨텍스트 스위칭 최소화: 하루 1~2개 작업만 집중
- [ ] "빠른 성공 리스트": 소소한 달성 항목 시각화
- [ ] 창의적 작업 ↔ 기계적 작업 번갈아 배치

### 개인 워크플로우 최적화 (1주)
- [ ] 일과 시작 전 30분: 중요/긴급 매트릭스로 우선순위
- [ ] 디버깅 전 15분: 문제 정의 문서화 (무작정 디버깅 금지)
- [ ] 금요일 회고: 이번 주 학습 내용 PermanentNote화
- [ ] 주 1회 "AI 없이 코딩" 날 (기본기 유지)

### 커리어 장기 전략 (3개월~1년)
- [ ] 성장 학습 80%: 원리(알고리즘, 설계 패턴, 웹 표준)
- [ ] 생존 학습 20%: 최신 프레임워크(React 19, Astro 등)
- [ ] AI 도구 숙련: Claude Code, Copilot CLI 활용
- [ ] 사이드 프로젝트: 의사결정/설계 역량 강화
- [ ] 커뮤니티 참여: 오픈소스 기여, 블로그 작성

### 팀/조직 개선 (가능 시)
- [ ] 코드 리뷰 문화: AI 리뷰(CodeRabbit) + 인간 전략적 리뷰
- [ ] 기술 부채 정리 시간: 스프린트의 20% 할당
- [ ] 회의 최소화: 비동기 커뮤니케이션 원칙
- [ ] 온콜 로테이션: 번아웃 방지
- [ ] 학습 문화: 사내 스터디, 컨퍼런스 참가 지원
